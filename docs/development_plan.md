# 开发计划

## 阶段划分

### 阶段1: 基础框架（Agent 1负责）
**时间**: 1-2天
**任务**:
- 数据包封装与解析 (packet.py)
- 校验和计算 (checksum.py)
- 日志工具 (logger.py)
- 定时器 (timer.py)

**交付物**:
- 完整的Packet类
- 工具函数库
- 单元测试

### 阶段2: 连接管理（Agent 2负责）
**时间**: 2-3天
**任务**:
- 连接状态机 (connection.py)
- 三次握手实现
- 四次挥手实现
- 连接超时处理

**交付物**:
- Connection类
- 握手/挥手测试用例

### 阶段3: 可靠传输（Agent 3负责）
**时间**: 3-4天
**任务**:
- 发送端逻辑 (sender.py)
- 接收端逻辑 (receiver.py)
- 滑动窗口实现
- 超时重传机制
- 累积确认

**交付物**:
- Sender和Receiver类
- 可靠传输测试

### 阶段4: 流量与拥塞控制（Agent 4负责）
**时间**: 2-3天
**任务**:
- 流量控制实现
- 拥塞控制算法 (congestion.py)
- 慢启动
- 拥塞避免
- 快速重传与快速恢复

**交付物**:
- CongestionControl类
- 拥塞控制测试

### 阶段5: 客户端与服务器（Agent 5负责）
**时间**: 2天
**任务**:
- 客户端主程序 (client.py)
- 服务器主程序 (server.py)
- 命令行参数解析
- 文件传输功能
- 日志输出

**交付物**:
- 完整的客户端和服务器程序
- 使用说明

### 阶段6: 测试与优化（所有Agent协作）
**时间**: 2-3天
**任务**:
- 集成测试
- 性能测试
- 丢包模拟测试
- Bug修复
- 代码优化
- 文档完善

**交付物**:
- 完整测试报告
- 最终版本代码

## Agent分工

| Agent | 负责模块 | 主要任务 |
|-------|----------|----------|
| Agent 1 | 基础框架 | packet, utils |
| Agent 2 | 连接管理 | connection, 握手挥手 |
| Agent 3 | 可靠传输 | sender, receiver, 窗口 |
| Agent 4 | 拥塞控制 | congestion, 流量控制 |
| Agent 5 | 应用层 | client, server |

## 开发规范

1. 代码风格: PEP 8
2. 注释: 关键函数必须有docstring
3. 日志: 使用统一的logger
4. 测试: 每个模块都要有单元测试
5. 版本控制: Git提交信息清晰
